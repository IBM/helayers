21 
22  === Loading NN architecture from Keras-generated JSON file ===
23 Below are instructions for how to use Keras to define a model with architecture
24 that's being supported by Helayers.
25 
26 * Using both the "Sequential" and "Functional" APIs to define the model
27 architecture is supported.
28 * The supported layers are "Conv2D", "AveragePooling2D", "Dense",
29 "SquareActivation", "PolyActivation", "Flatten", "Input", "Add", "Concatenate"
30 and "BatchNormalization".
31 * The NN can be non-sequential: layer's output can be the input of multiple
32 layers, and the outputs of two layers can be added or concatenated to one
33 another as long as the shapes are compatible. The layers must form a DAG, where
34 all paths start and end in the same nodes.
35 * The NN must start with either a Conv2D or a Dense layer, not including an
36 Input layer that must be specified, either as a seperate layer or not.
37 * If both Conv2D/AveragePooling2D and Dense layers are present in a path along
38 the NN, all Conv2D/AveragePooling2D layers must come before any Dense layer, and
39 a Flatten layer must be present after the last Conv2D/AveragePooling2D layer and
40 before the first Dense layer. Between the Flatten layer and the first Dense
41 layer that follows, the path must remain sequential.
42 * Flatten layer cannot be present in any case other than having both
43 Conv2D/AveragePooling2D and Dense layers in a path along the NN.
44 * SquareActivation and PolyActivation layer must be specified as a separate
45 layer and may be present at any location other than being the first layer of the
46 NN.
47 * BatchNormalization layer must be applied with respect to the channels
48 dimension and must therefore be present only in NNs with Conv2D layer(s), and
49 not after any Flatten layer.
50 
51 *** Conv2D ***
52 The supported attributes are: (any other attribute will be ignored)
53  "filters" - required
54  "kernel_size" - required
55  "strides" - required
56  "padding" - required, either "valid" or "same".
57  "batch_input_shape" - structure: (height, width, channels), required if first
58  layer and no input layer was explicitly added before, will be ignored
59  otherwise
60  "name" - optional
61 
62 *** AveragePooling2D ***
63 The supported attributes are: (any other attribute will be ignored)
64  "pool_size" - required
65  "strides" - required
66  "padding" - required, either "valid" or "same".
67  "name" - optional
68 NOTICE: when an average pooling layer includes padding, its evaluation by
69 Helayers will include the pads when averaging, regardless of any setting or
70 behaviour of an external library.
71 
72 *** Dense ***
73 The supported attributes are: (any other attribute will be ignored)
74  "units" - required
75  "batch_input_shape" - structure: (height,), required if first layer and no
76  input layer was explicitly added before, will be ignored otherwise
77  "name" - optional
78 
79 *** SquareActivation ***
80 The supported attributes are: (any other attribute will be ignored)
81  "name" - optional
82 
83 *** PolyActivation ***
84 The supported attributes are: (any other attribute will be ignored)
85  "coefs" - required, an array with the coefficients for the poly activation.
86  The first coefficient corresponds to the highest degree (for example, [1,
87  -32.5, 8.79] will be evaluated as x^2 - 32.5x + 8.79).
88  "name" - optional
89 
90 *** Flatten ***
91 The supported attributes are: (any other attribute will be ignored)
92  "name" - optional
93 
94 *** Input ***
95 The supported attributes are: (any other attribute will be ignored)
96  "name" - optional
97 
98 *** Add ***
99 The supported attributes are: (any other attribute will be ignored)
100  "name" - optional
101 
102 *** Concatenate ***
103 The supported attributes are: (any other attribute will be ignored)
104  "axis" - required, the axis to concatenate along, one of 1,2,3. Axis 1, 2 and
105  3 will be interpreted as the rows, columns and channels dimensions,
106  respectively.
107  "name" - optional
108 
109 *** BatchNormalization ***
110 The supported attributes are: (any other attribute will be ignored)
111  "epsilon" - required, the epsilon value to use for the normalization, to avoid
112  deviding by zero.
113  "axis" - required, the axis of the batch normalization, must be 3 (the
114  channels dimensions).
115  "name" - optional
116 
117 */
