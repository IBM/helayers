Detailed Description

Loading NN architecture from Keras-generated JSON file

Below are instructions for how to use Keras to define a model with architecture that's being supported by Helayers.

  • Using both the "Sequential" and "Functional" APIs to define the model architecture is supported.
  • The supported layers are "Conv2D", "AveragePooling2D", "Dense", "SquareActivation", "PolyActivation", "Flatten", "Input", "Add", "Concatenate" and "BatchNormalization".
  • The NN can be non-sequential: layer's output can be the input of multiple layers, and the outputs of two layers can be added or concatenated to one another as long as the shapes are compatible. The layers must form a DAG, where all paths start and end in the same nodes.
  • The NN must start with either a Conv2D or a Dense layer, not including an Input layer that must be specified, either as a seperate layer or not.
  • If both Conv2D/AveragePooling2D and Dense layers are present in a path along the NN, all Conv2D/AveragePooling2D layers must come before any Dense layer, and a Flatten layer must be present after the last Conv2D/AveragePooling2D layer and before the first Dense layer. Between the Flatten layer and the first Dense layer that follows, the path must remain sequential.
  • Flatten layer cannot be present in any case other than having both Conv2D/AveragePooling2D and Dense layers in a path along the NN.
  • SquareActivation and PolyActivation layer must be specified as a separate layer and may be present at any location other than being the first layer of the NN.
  • BatchNormalization layer must be applied with respect to the channels dimension and must therefore be present only in NNs with Conv2D layer(s), and not after any Flatten layer.

Conv2D

  • The supported attributes are: (any other attribute will be ignored)
    • "filters" - required
    • "kernel_size" - required
    • "strides" - required
    • "padding" - required, either "valid" or "same".
    • "use_bias" - required, either "true" or "false".
    • "batch_input_shape" - structure: (height, width, channels), required if first layer and no input layer was explicitly added before, will be ignored otherwise
    • "name" - optional

AveragePooling2D

  • The supported attributes are: (any other attribute will be ignored)
    • "pool_size" - required
    • "strides" - required
    • "padding" - required, either "valid" or "same".
    • "name" - optional
  • NOTICE: when an average pooling layer includes padding, its evaluation by Helayers will include the pads when averaging, regardless of any setting or behaviour of an external library.

Dense

  • The supported attributes are: (any other attribute will be ignored)
    • "units" - required
    • "use_bias" - required, either "true" or "false".
    • "batch_input_shape" - structure: (height,), required if first layer and no input layer was explicitly added before, will be ignored otherwise
    • "name" - optional

SquareActivation

  • The supported attributes are: (any other attribute will be ignored)
    • "name" - optional

PolyActivation

  • The supported attributes are: (any other attribute will be ignored)
    • "coefs" - required, an array with the coefficients for the poly activation. The first coefficient corresponds to the highest degree (for example, [1, -32.5, 8.79] will be evaluated as x^2 - 32.5x + 8.79).
    • "name" - optional

Flatten

  • The supported attributes are: (any other attribute will be ignored)
    • "name" - optional

Input

  • The supported attributes are: (any other attribute will be ignored)
    • "name" - optional

Add

  • The supported attributes are: (any other attribute will be ignored)
    • "name" - optional

Concatenate

  • The supported attributes are: (any other attribute will be ignored)
    • "axis" - required, the axis to concatenate along, one of 1,2,3. Axis 1, 2 and 3 will be interpreted as the rows, columns and channels dimensions, respectively.
    • "name" - optional

BatchNormalization

  • The supported attributes are: (any other attribute will be ignored)
    • "epsilon" - required, the epsilon value to use for the normalization, to avoid deviding by zero.
    • "axis" - required, the axis of the batch normalization, must be 3 (the channels dimensions).
    • "name" - optional
